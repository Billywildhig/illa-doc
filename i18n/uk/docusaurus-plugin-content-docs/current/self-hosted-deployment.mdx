# Розгортання власного хостингу

## Докер

1.  Запустити Docker
2.  Введіть наступну команду в терміналі для встановлення офіційного зображення:

`docker pull illasoft/illa-builder:latest`

## КЛІ

### Завантажити ILLA CLI та розгорнути

Якщо у вас встановлена Rust, ви можете встановити останній випуск ILLA CLI за допомогою cargo:

`cargo install illa`

`illa розгортання --self --port=10000`

Інструмент **авто** буде завершити завантаження ILLA CLI його вихідними залежностями, побудувати і встановити його в шлях вантажівки, щоб ми могли його запустити. Після установки ви можете запустити ILLA CLI за допомогою **illa** команди.

Крім того, ви можете просто завантажити ILLA CLI для вашої операційної системи тут:

-   **[Windows](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-win.zip)**
-   **[Linux](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar.gz)**
-   **[Mac](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-macos.tar.gz)**

Якщо ви завантажуєте з вищевказаних посилань, кроки трошки відрізняються в порівнянні з завантаженням ILLA CLI з **вантажити** . Після завантаження інтерфейсу користувача, вам потрібно буде його виконувати. Давайте візьмемо ілюстраційний штрих, наприклад:

```Plaintext
# download illa cli binary file
> wget <https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar.gz>

# unpack the file
> tar -zxvf illa-x86_64-linux.tar.gz

> cd illa-x86_64-linux

> chmod +x illa

# run the illa cli
> ./illa
```

### Перевірка передумов встановлення власного хостингу

Ці порти 5432, 9999 та 8000 повинні залишатися в конструкторі ILLA

Наступна команда перевіряє передумови встановлення на вашій операційній системі:

```Plaintext
> illa doctor
```

### Розгортання ILLA Builder

Після встановлення ILLA CLI, ви можете запустити встановлення на свій хостинг або хмаринкового встановлення.

Ви можете просто розгорнути ILA Builder для самостійного хостингу, запустивши наступну команду:

```Plaintext
# the port which ILLA Builder can be accessed on can be changed
# the <SERVER_ADDRESS> should be your cloud server public ip, default localhost
> illa deploy --self --port=10000 --server-addr=<SERVER_ADDRESS>
```

Тепер ви можете отримати доступ до ILLA Builder: <http://localhost:10000>

### Оновлення ILLA Builder

Щоб оновити Конструктор ILLA з останніми зображеннями докерів, використовуйте наступну команду:

```Plaintext
> лла оновлення --self
```

### Інші операції

```Plaintext
# list the ILLA Builder
> illa list --self

# stop the ILLA Builder
> illa stop --self

# remove the ILLA Builder
> illa remove --self

# forced removal of the running ILLA Builder and persistent data
> sudo illa remove --force --self --data

# restart the ILLA Builder
> illa restart --self

# help information
> illa help
```

## Kubernetes

### Опис

Сконструюйте усі зображення в одному і виконайте його на вашому комп’ютері. You can check out the scripts file in the [scripts](https://github.com/illacloud/deploy-illa-manually/blob/main/kubernetes/scripts) folder for more details.

Нотатка:

We highly recommended deploying with our auto-deploy tools [illa-cli](https://github.com/illacloud/illa).

І на даний момент ми не підтримуємо арх Apple Silicon M1 (darwin-6).

### Запустити з офіційним зображенням Slim

Встановити GNU значення і тип:

```JavaScript
зробити розгортання
```

або просто виконати:

```JavaScript
/bin/bash скрипти/розгортання
```

ця команда буде витягувати illasoft офіційний весь в-одному зображенні і розгорнути його на вашому кластері Kubernetes.

### Для постійного сховища бази даних

Редагувати [illa-builder.yaml](https://github.com/illacloud/deploy-illa-manually/blob/main/kubernetes/illa-builder.yaml) і додати вашу IAAS збережену конфігурацію сховища.

### Для конфігурації HTTPS

Ви можете маршрутизувати вузловий порт вашого Kubernetes-кластера і переписати до 443 порту, а також додати https cert в шлюзі ingress

Або ви можете розмістити термостат ingress вручну до вашого сервера Kubernetes, конфігурація як тут:

```JavaScript
static_resources:
  listeners:
  - name: https_listener
    address:
      socket_address:
        address: 0.0.0.0
        port_value: 443
    filter_chains:
    - filters:
      - name: envoy.filters.network.http_connection_manager
        typed_config:
          "@type": type.googleapis.com/envoy.extensions.filters.network.http_connection_manager.v3.HttpConnectionManager
          stat_prefix: https_listener
          route_config:
            name: local_route
            virtual_hosts:
            - name: illa_builder
              domains:
              - "illa.yourdomian.com" # replace with your domain
              routes:
              - match:
                  prefix: "/"route:
                  cluster: illa_builder
          http_filters:
          - name: envoy.filters.http.router
            typed_config:
              "@type": type.googleapis.com/envoy.extensions.filters.http.router.v3.Router
      transport_socket:
        name: envoy.transport_sockets.tls
        typed_config:
          "@type": type.googleapis.com/envoy.extensions.transport_sockets.tls.v3.DownstreamTlsContext
          common_tls_context:
            tls_certificates:
            # replace this with your cert file
            - certificate_chain:
                filename: /your-cert-folder/fullchain.pem
              private_key:
                filename: /your-cert-folder/privkey.pem

  clusters:
  - name: illa_builder
    type: STRICT_DNS
    lb_policy: ROUND_ROBIN
    connect_timeout: 10s
    load_assignment:
      cluster_name: illa_builder
      endpoints:
      - lb_endpoints:
        - endpoint:
            address:
              socket_address:
                address: illa-builder
                port_value: 80
```

## Розгортання Quickly

Ви можете дуже швидко розгорнути ILLA, будь ласка, дотримуйтесь інструкцій нижче:

Спочатку, будь ласка, завантажте цей файл [docker](https://github.com/illacloud/deploy-illa-manually/tree/main/docker) на ваш комп'ютер, а потім запустіть наступний код.

```JavaScript
cd docker;
/bin/bash ./scripts/run-official-image.sh;
```

І увійдіть за допомогою імені користувача і пароля за замовчуванням:

Ім'я користувача: **`root`**

Пароль: **`пароль`** (тільки для власного хосту).
