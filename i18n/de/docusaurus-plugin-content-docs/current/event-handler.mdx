## Konfigurationsoptionen

### Ereignis

Wird verwendet, um die Aktion auszulösen. Zum Beispiel, wenn wir restapi1 auslösen wollen, wenn die Datenraster(Tabelle) Komponente aktualisiert wird, können wir das Ereignis auf `Aktualisieren` setzen. Wenn wir restapi2 auslösen wollen, wenn die Art der Daten-Grid(Tabelle) Komponente geändert wird, können wir das Ereignis so einstellen, dass es Änderungen sortiert. Wenn wir restapi3 auslösen wollen, wenn der Button geklickt wird, können wir das Ereignis auf `click` setzen.

#### In ILLA Laufwerk speichern

##### Parameter

`Dateiname mit Erweiterung`: String. Name mit Endung der Datei nach dem Hochladen auf ILLA Drive.

`Datei Daten`: String. Base64 oder Rohdaten wie z.B. JSON, CSV, Klartext, etc.

`Dateityp`: String. Einer der folgenden Werte: Auto, Plain text, JPEG, PNG, SVG, JSON, CSV, TSV, Excel(.xlsx). Bei Verwendung von "Auto" werden wir versuchen, den Dateityp anhand der Dateierweiterung oder der Datei-Daten zu ermitteln.

`Ordner`: String. Zielordner für das Hochladen von Dateien. Leer lassen um in den Root-Ordner hochzuladen.

"Anonym erlauben": Bolean. Wird verwendet, um festzulegen, ob anonymer Benutzerzugriff erlaubt ist. Wenn Sie Ihre Anwendung als öffentlich einstellen wollen, müssen anonyme Zugriffe aktiviert sein. Zur Sicherheit Ihrer Daten müssen Dateien oder Ordner anonym im anonymen Ordner abgelegt werden.

`Ersetzen`: Boolean. Wird zum Setzen verwendet, wenn Dateien mit dem gleichen Namen im selben Verzeichnis vorhanden sind, ersetzen Sie die vorhandene Datei durch eine neue Datei oder benennen Sie die neue Datei automatisch um.

### Nur ausführen, wenn

Ein boolescher Wert, der verwendet wird, wenn der Ereignishandler ausgelöst werden konnte. Wenn beispielsweise die Eingabe 1 null ist, konnte der Ereignishandler nicht ausgelöst werden: `{{input1.value!==""&&input1.value!==undefined}}`
