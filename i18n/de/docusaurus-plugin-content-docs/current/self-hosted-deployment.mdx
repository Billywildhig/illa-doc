# Selbstgehosteter Einsatz

## Docker

1.  Docker starten
2.  Geben Sie den folgenden Befehl im Terminal ein, um das offizielle Image zu installieren:

`docker ziehen illasoft/illa-builder:latest`

## CLI

### Lade ILLA CLI herunter und verteilen

Wenn Sie Rust installiert haben, können Sie ganz einfach die neueste ILLA CLI Version mit Ladung installieren:

`cargo install illa`

`illa Deploy --self --port=10000`

Das **Cargo** Tool wird das Herunterladen des ILLA CLI mit seinen Quell-Abhängigkeiten beenden bauen und installieren Sie es in den Frachtschrank-Pfad, so dass wir ihn ausführen können. Nach der Installation können Sie das ILLA CLI mit dem **illa** Befehl ausführen.

Alternativ können Sie auch den ILLA CLI für Ihr Betriebssystem hier herunterladen:

-   **[Windows](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-win.zip)**
-   **[Linux](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar.gz)**
-   **[Mac](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-macos.tar.gz)**

Wenn Sie von den obigen Links herunterladen, sind die Schritte etwas anders als das Herunterladen des ILLA CLI von **cargo** . Nach dem Download des CLI müssen Sie ihn ausführbar machen. Nehmen wir das illa-linux zum Beispiel:

```Plaintext
# download illa cli binary file
> wget <https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar.gz>

# unpack the file
> tar -zxvf illa-x86_64-linux.tar.gz

> cd illa-x86_64-linux

> chmod +x illa

# run the illa cli
> ./illa
```

### Überprüfung der Voraussetzungen für selbstgehostete Installation

Diese Ports 5432, 9999 und 8000 sollten beim ILLA Builder verbleiben

Der folgende Befehl prüft die Voraussetzungen für die selbstgehostete Installation auf Ihrem Betriebssystem:

```Plaintext
> illa doctor
```

### ILLA-Builder wird eingesetzt

Sobald die ILLA CLI installiert ist, können Sie eine selbst gehostete oder Cloud-Installation durchführen.

Sie können einfach einen selbst gehosteten ILA Builder bereitstellen, indem Sie den folgenden Befehl ausführen:

```Plaintext
# der Port, auf den der ILLA Builder zugegriffen werden kann, kann geändert werden
# die <SERVER_ADDRESS> sollte Ihre öffentliche Cloud-Server-IP sein, default localhost
> illa Deploy --self --port=10000 --server-addr=<SERVER_ADDRESS>
```

Jetzt kannst du auf den ILLA Builder zugreifen: <http://localhost:10000>

### Aktualisiere ILLA Builder

Um den ILLA Builder mit den neuesten Docker-Bildern zu aktualisieren, verwenden Sie folgenden Befehl:

```Plaintext
> illa update --self
```

### Andere Operationen

```Plaintext
# den ILLA Builder
> illa list --self

# den ILLA Builder
> illa stop --self

# Entfernen Sie den ILLA Builder
> illa remove --self

# erzwungene Entfernung des ILLA Builders und der persistenten Daten
> sudo illa remove --force --self --data

# Neustart des ILLA Builders
> illa Neustart --self

# Hilfe
> illa Hilfe 
 > illa
```

## Kubernetes

### Beschreibung

Erstellen Sie illa All-in-One Image und führen Sie es mit k8s auf Ihrem Rechner aus. Du kannst die Skript-Datei im Ordner [scripts](https://github.com/illacloud/deploy-illa-manually/blob/main/kubernetes/scripts) für weitere Details sehen.

Hinweis:

Wir empfehlen dringend den Einsatz mit unseren Auto-Deploy-Tools [illa-cli](https://github.com/illacloud/illa).

Und im Moment unterstützen wir nicht Apple Silicon M1 (darwin-arm64 Bogen).

### Mit einem offiziellen Slim Bild ausführen

GNU make installieren und tippen:

```JavaScript
bereitstellen
```

oder einfach ausführen:

```JavaScript
/bin/bash scripts/deploy.sh
```

mit diesem Befehl wird illasoft offizielles All-In-One-Bild gezogen und in Ihrem Kubernetes Cluster eingesetzt.

### Für den Persistenten Speicher der Datenbank

Bearbeiten Sie [illa-builder.yaml](https://github.com/illacloud/deploy-illa-manually/blob/main/kubernetes/illa-builder.yaml) und fügen Sie Ihre IAAS persistente Speicherkonfiguration hinzu.

## Schnell bereitstellen

Sie können ILLA auf bemerkenswert schnelle Weise einsetzen, folgen Sie bitte den folgenden Anweisungen:

Bitte lade zuerst diese [docker file](https://github.com/illacloud/deploy-illa-manually/tree/main/docker) auf deinen Computer herunter und führe den folgenden Code aus.

```JavaScript
cd Docker;
/bin/bash ./scripts/run-official-image.sh;
```

Und melden Sie sich mit dem Standardnamen und Passwort an:

Benutzername: **`root`**

Passwort: **`password`** (nur Self-Host-Modus).
