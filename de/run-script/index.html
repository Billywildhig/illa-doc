setFilters(Array von Objekten, Zeichenkette)​

Zum Setzen von Filtern verwendet.

Der erste Parameter ist ein Array von Objekten. Jedes Objekt stellt eine Filterbedingung dar. Er enthält drei Attribute:

id : Dies bezieht sich auf die Id dieser Spalte. Wenn diese Spalte ein Ergebnis einer Datenbankabfrage ist, wäre die ID der Spaltenname, den Sie aus der Abfrage geholt haben. Zum Beispiel, wenn die Abfrage SELECT email AS user_email FROM users ist und email in Email umbenennt, indem das rechte Panel der Tabelle verwendet wird die ID der Spalte wäre immer noch user_email wenn diese Spalte manuell mit der Schaltfläche Spalte hinzufügen in der rechten Hand hinzugefügt wird, du kannst seine ID im Abschnitt displayedData des linken Fensters überprüfen. Normalerweise wird es eine zufällig generierte Zeichenkette sein, die mit column beginnt, wie column-320169e8-225a-482d-9853-ad34b3040220.

filterFn : der Match-Typ des Filters. Wir unterstützen die folgenden Typen: equalTo, was die gleiche ist wie = in SQL, notEqualTo ist identisch mit <> in SQL, contains und entspricht dem LIKE '%value%' in SQL, does NotContain was das gleiche ist wie NOT LIKE '%value%' in SQL, lessThan ist identisch mit < in SQL, notLessThan, was das gleiche ist wie >= in SQL, notMoreThan ist identisch mit <= in SQL, empty was das gleiche ist wie IS NULL in SQL, notEmpty ist das gleiche wie IST NICHT NULL in SQL, before wird verwendet, um Zeit oder Datum zu vergleichen, was das gleiche ist wie > in SQL, after wird verwendet, um Zeit oder Datum zu vergleichen, was dasselbe ist wie < in SQL.

value : wird verwendet, um den zu vergleichenden Wert zu setzen.

Der zweite Parameter: Wenn es mehrere Filterbedingungsobjekte gibt, bestimmt er, ob sie eine "AND" Beziehung oder eine "ODER" Beziehung haben. Setze es als String and für eine "AND"-Beziehung, und oder für eine "OR"-Beziehung.

Unterstützte Komponenten​

Tisch

Beispiel​

In ILLA Laufwerk speichern​

Struktur​

utils.saveToILLADrive({
fileName:string,
fileData:string,
fileType:string,
folder:string,
allowAnonymous:boolean,
replace:boolean
})

Beschreibung​

Wird verwendet, um eine Datei im ILLA Laufwerk zu speichern.

fileName: Name mit Erweiterung der Datei nach dem Hochladen in ILLA Drive.

fileData: Base64 oder Rohdateien, wie JSON, CSV, Klartext, etc.

fileType: Einer der folgenden Werte: Auto, Plain text, JPEG, PNG, SVG, JSON, CSV, TSV, Excel(.xlsx). Bei Verwendung von "Auto" werden wir versuchen, den Dateityp anhand der Dateierweiterung oder der Datei-Daten zu ermitteln.

Ordner: Zielordner für Datei-Upload. Leer lassen um in den Root-Ordner hochzuladen.

'allowAnonymous': Wird benutzt um festzulegen, ob anonymer Benutzerzugriff erlaubt ist. Wenn Sie Ihre Anwendung als öffentlich einstellen wollen, müssen anonyme Zugriffe aktiviert sein. Zur Sicherheit Ihrer Daten müssen Dateien oder Ordner anonym im anonymen Ordner abgelegt werden.

replace: Wird zum Setzen verwendet, wenn es Dateien mit dem gleichen Namen im selben Verzeichnis gibt, ersetzen Sie die vorhandene Datei durch eine neue Datei oder benennen Sie die neue Datei automatisch um.

Beispiel​

//upload1 verwenden, um Dateien vom Gerät auszuwählen und in den App-Ordner hochzuladen. 
utils.saveToILLADrive({
fileName:upload1.files[0].name,
fileData:upload1. iles[0].dataURI,
fileType:"PNG",
Ordner:"apps",
allowAnonymous:false,
ersetzen: true
})
//Laden Sie eine Textdatei hoch.  
utils.saveToILLADrive({
fileName:"test.txt",
fileData:"abc",
fileType:"Plain text",
Ordner:"apps",
allowAnonym:false,
replace:true
})

Beschreibung​

Wird verwendet, um Daten auf Ihr Gerät zu exportieren.

fileType: Einer der folgenden Werte: Auto, Plain text, JPEG, PNG, SVG, JSON, CSV, TSV, Excel(.xlsx). Bei Verwendung von "Auto" werden wir versuchen, den Dateityp anhand der Dateierweiterung oder der Datei-Daten zu ermitteln.

fileName: ein Name mit Erweiterung.

data: Daten der Datei.

Beispiel​

//Laden Sie die Daten der Tabelle als CSV
utils.downloadFile({
    fileType:"CSV",
    fileName:"text.csv",
    data: table2.dataSourceJS
})
//Bild
utils.downloadFile({
    fileType:"PNG",
    fileName:"text.png",
    data: image1.imageSrc
})

Globale Daten festlegen​

Globalen Datenwert festlegen​

Struktur​
utils.setGlobalDataValue({
key:string, 
value:any
})
Beschreibung​

Verwendet um den Wert von gobalData zu setzen

key: der displayName von globalData wie der folgende Screenshot zeigt.

value: der zu setzende Wert

Beispiel​
//verwende input5, um den Benutzereingabewert zu erhalten und als Wert von stringState zu setzen. 
utils.setGlobalDataValue({
key:"stringState", 
value:input5.value
})

Globale Daten setzen in​

Struktur​
utils.setGlobalDataIn({
key:string, 
path:string, 
value:any
})
Beschreibung​

Wenn die globalen Daten ein Array oder Objekt sind, setGlobalDataIn verwenden, um den Wert zu ändern, der einem bestimmten Index in einem Array entspricht, oder um den Wert zu ändern, der einem bestimmten Schlüssel in einem Objekt entspricht.

key: der displayName von globalData.

path: der Index des Wertes, den Sie im Array oder dem Schlüssel des Objekts aktualisieren wollen. Zum Beispiel, um den zweiten Wert im Array zu setzen, ist der Pfad 1 (der Index). Um den Wert des Attributs Key1 zu setzen, ist der Pfad key1.

value: der zu setzende Wert

Beispiel​
//objectState ist ein Objekt: {"key1":"value1", "key2":"value2"}, um die 
utils.setGlobalDataIn({
key:"objectState", 
path:"key2", 
value:input7.value
})

Wert der Aktionen, Komponenten...​

Benutze displayName.xxx um den Wert von Aktionen, Komponenten oder anderen globalen Daten zu verwenden. Bitte beachten Sie, dass diese Variablen im Ausführungsskript ohne {{ Referenzen direkt aufgerufen werden können.

utils.downloadFromILLADrive({
    downloadInfo:{{drivePicker1.value}},//❌ 
    asZip:true
})

utils.downloadFromILLADrive({
    downloadInfo:drivePicker1.value,//:check_mark_button:
    asZip:true
})
